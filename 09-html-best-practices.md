# 🎬 9. HTML Best Practices (Write Clean & Professional Code)

## 🚀 What are Best Practices?

Best practices are **standard rules** that make your code:

* Clean 🧹
* Readable 👀
* Maintainable 🔧
* Professional 💻

👉 Good code is not just working code — it’s **clean code**

---

## 🧠 Real-Life Analogy

Think of writing notes:

* Messy notes ❌ → Hard to understand
* Clean notes ✅ → Easy to revise

👉 Same with HTML

---

## 🧱 1. Clean Structure

Always follow proper HTML structure:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>My Page</title>
  </head>
  <body>
    <h1>Hello</h1>
    <p>Welcome to my website</p>
  </body>
</html>
```

### ⚡ Why it matters

* Browser reads correctly
* Easy debugging
* Looks professional

---

## 📐 2. Proper Indentation

Indent your code for readability:

### ❌ Bad Code

```html
<html><body><h1>Hello</h1><p>Text</p></body></html>
```

---

### ✅ Good Code

```html
<html>
  <body>
    <h1>Hello</h1>
    <p>Text</p>
  </body>
</html>
```

### ⚡ Rule

👉 Use **2 spaces or 4 spaces consistently**

---

## 🏷️ 3. Meaningful Tags (Semantic HTML)

Use tags that describe content:

### ❌ Avoid

```html
<div>Header</div>
<div>Content</div>
```

---

### ✅ Use

```html
<header>Header</header>
<article>Content</article>
```

### ⚡ Why it matters

* Better SEO 📈
* Better accessibility ♿
* Cleaner structure

---

## 🔍 Bonus Best Practices (Important 🔥)

### ✔ Always Close Tags

```html
<p>Text</p>
```

---

### ✔ Use Lowercase Tags

```html
<h1>Correct</h1>
```

---

### ✔ Add `alt` for Images

```html
<img src="img.jpg" alt="Description">
```

---

### ✔ Keep File Names Clean

```
about.html  ✅
About Page.html ❌
```

---

## ❌ Common Beginner Mistakes

* No indentation ❌
* Too many `<div>` ❌
* Random structure ❌
* Ignoring semantic tags ❌
